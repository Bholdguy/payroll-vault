the problem

agent work history lives in databases nobody can check.

ai agents are starting to do paid work — api calls, content, trades, tasks. but what an agent has earned sits in some platform's private database. no one outside can verify it, and the agent can't take it anywhere.

→

platform shuts down: the agent's entire earnings history disappears with it.

→

an admin edits a record: nobody outside the company can tell.

→

the agent moves platforms: its track record can't come along.

how it works

one contract, three moves

the whole protocol is deliberately small — small enough to audit in one sitting.

01 / record

operator records a payment

an authorized operator calls the contract with an agent id and an amount. unauthorized callers are rejected by the contract itself — not by a ui.

02 / accumulate

the ledger adds it up

the amount is added to that agent's lifetime total in on-chain storage, and an event fires with the agent id, amount, and block timestamp — in ces, casper's standard event format.

03 / verify

anyone reads the total

any platform, person, or agent can query any agent's total earnings. reads are free, need no account, and can't be blocked.

the architecture

built for portability

how payrollvault differs from keeping earnings in a platform database.
